i am building a project for management of projects and clients, and i have a problem with a certain feature. Project can be of type regular, or retainer. Retainer projects are recurring projects, which means they are eternally on my schedule, cycling over and over. this kind of project also can have recurring steps (some kind of tasks) that can also be recuring. A recurring step can recur every amount of months, days, or weeks with specific days in the week. On the schedule i can also see future occurrences of the step so i will know for every step when it will be in the future. here are some examples:
1) i have a step that occur every 5 days. i will see it on the schedule, and i will complete the step. the step will have its property "isComplete" with value true, and when its dateOnWeekly will come it will automatically change isComplete to false.
2) i have a step the occur every Sunday and Tuesday. i will be able to complete the next occurrence, and once this date will come the step will change its isComplete back to false
3) Suppose:
RecurringDateType = week
RecurringEvery = 2 (every 2 weeks)
RecurringDaysInWeek = [0, 2] (Sunday, Tuesday)
then Every Sunday and every Tuesday, but the whole cycle is considered “every 2 weeks” in some other sense.
4) For RecurringDateType = month i want to use the same calendar day, and not the weekday. in cases that there is no such a day in the month (like 30th February) then use the last day of the month instead.

the future occurrences are like ghosts, or temporary steps. i generate them dynamically in the front to give the user indication, but the are not interactable. 
in the server i have this code that check weather a recurring step should change its isComplete from true to false:

private void UpdateRetainerSteps(List<Step> steps)
{
    foreach (Step step in steps)
    {
        if (step.IsRecurring)
        {
            DateTime nextOccurrence = Functions.SetDefaultHHMMSS(step.NextOccurrence ?? step.DateCreated ?? new DateTime(0));
            var today = Functions.SetDefaultHHMMSS(DateTime.UtcNow);

            if (nextOccurrence <= today && step.IsComplete)
            {
                step.IsComplete = false;
                step.DateCompleted = null;
                step.PositionInWeeklyList = 9999;

                // Handle weekly recurrence with specific days in week
                if (step.RecurringDateType == RecurringDateTypeEnum.week)
                {
                    // RecurringDaysInWeek expected to be ints matching DayOfWeek (0 = Sunday ... 6 = Saturday)
                    if (step.RecurringDaysInWeek != null && step.RecurringDaysInWeek.Count > 0)
                    {
                        var days = step.RecurringDaysInWeek.Distinct().OrderBy(d => d).ToList();

                        // Advance nextOccurrence to the next matching weekday (taking RecurringEvery into account)
                        while (nextOccurrence <= today || (step.FutureModifiedTasks != null && step.FutureModifiedTasks.Contains(nextOccurrence)))
                        {
                            int currentDay = (int)nextOccurrence.DayOfWeek;
                            // find next day in the same week strictly after current day
                            int? nextDayInList = days.FirstOrDefault(d => d > currentDay);
                            DateTime candidate;

                            if (nextDayInList != null && nextDayInList > currentDay)
                            {
                                int diff = nextDayInList.Value - currentDay;
                                candidate = nextOccurrence.AddDays(diff);
                            }
                            else
                            {
                                // no later day in this week -> jump to the next recurrence week block and pick the first configured weekday
                                int weeksToAdd = step.RecurringEvery ?? 1;
                                var startOfNextBlock = nextOccurrence.AddDays(7 * weeksToAdd);
                                int firstDay = days.First();
                                int diff = (firstDay - (int)startOfNextBlock.DayOfWeek + 7) % 7;
                                candidate = startOfNextBlock.AddDays(diff);
                            }

                            nextOccurrence = Functions.SetDefaultHHMMSS(candidate);
                        }

                        step.NextOccurrence = nextOccurrence;
                        // we've already advanced past today for the weekly-with-specific-days case
                        continue;
                    }
                    // else: no specific days configured - fall through to the generic weekly handling below
                }

                // Generic advancement for day/week/month (covers weekly when no specific weekdays specified)
                while (nextOccurrence <= today)
                {
                    step.DateOnWeekly = nextOccurrence;
                    switch (step.RecurringDateType)
                    {
                        case RecurringDateTypeEnum.day:
                            nextOccurrence = nextOccurrence.AddDays(step.RecurringEvery ?? 1);
                            break;
                        case RecurringDateTypeEnum.week:
                            nextOccurrence = nextOccurrence.AddDays(7 * (step.RecurringEvery ?? 1));
                            break;
                        case RecurringDateTypeEnum.month:
                            nextOccurrence = nextOccurrence.AddMonths(step.RecurringEvery ?? 1);
                            break;
                    }
                }

                step.DateCreated = step.NextOccurrence;
                step.NextOccurrence = nextOccurrence;
            }
        }
    }
}
in this logic, the future completed step should be "reloaded", to indicate that it is no longer a future completed step and now it should be shown as not completed. this should happen at the moment that today catch up with the step date.
